# Подключение Google Sheets

Эта инструкция включает синхронизацию прогресса между телефоном и компьютером через Google Таблицу.

## 1. Создайте таблицу

1. Откройте Google Sheets.
2. Создайте новую таблицу, например `Читательский дневник Кати`.
3. Можно оставить первый лист пустым. Скрипт сам создаст лист `progress` и заголовки.

## 2. Вставьте Apps Script

1. В Google Таблице откройте `Расширения -> Apps Script`.
2. Удалите весь примерный код в редакторе.
3. Скопируйте содержимое файла `google-apps-script.js` из этого проекта.
4. Вставьте его в редактор Apps Script.
5. Нажмите `Сохранить`.

## 3. Опубликуйте Web App

1. Нажмите `Развернуть -> Новое развертывание`.
2. В типе выберите `Веб-приложение`.
3. В поле `Выполнять от имени` выберите `Я`.
4. В поле `Кто имеет доступ` выберите `Все, у кого есть ссылка`.
5. Нажмите `Развернуть`.
6. Google попросит разрешения. Подтвердите доступ к таблице.
7. Скопируйте URL веб-приложения. Он выглядит примерно так:

```text
https://script.google.com/macros/s/AKfycb.../exec
```

## 4. Вставьте URL в приложение

Откройте `script.js` и найдите строку:

```js
const CLOUD_API_URL = "";
```

Вставьте URL между кавычками:

```js
const CLOUD_API_URL = "https://script.google.com/macros/s/AKfycb.../exec";
```

После этого приложение будет:

- при открытии загружать прогресс из Google Таблицы;
- после сохранения книги отправлять прогресс в Google Таблицу;
- при сбросе прогресса очищать строки текущего ребенка в таблице;
- сохранять локальную копию в браузере как резерв.

## 5. Ссылка для Кати

Используйте длинный уникальный токен, а не короткое `katya`:

```text
index.html?child=katya-7f3a9c21-reading-map&name=Катя
```

Одна и та же ссылка должна открываться на телефоне и компьютере. Тогда оба устройства будут работать с одним прогрессом.

## 6. Проверка

1. Откройте приложение на компьютере по ссылке с `child=...`.
2. Отметьте одну книгу прочитанной.
3. Откройте Google Таблицу и проверьте лист `progress`.
4. Откройте ту же ссылку на телефоне.
5. Прогресс должен подтянуться после загрузки страницы.

## Важно

- Не публикуйте ссылку с `child=...` публично.
- Не храните в дневнике фамилию, школу, адрес и другие личные данные.
- Если вы меняете код Apps Script после публикации, нужно сделать новое развертывание или обновить существующее.
- Google Sheets подходит для домашнего проекта. Для большой публичной версии лучше Supabase или Firebase.
