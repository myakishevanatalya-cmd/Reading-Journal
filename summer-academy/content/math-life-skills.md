# Математика в жизни

## Раздел

Математика

## Блок

Величины, время, деньги, геометрия и закономерности.

## Возраст

Лето после 2 класса.

## Цель

Научить ребенка применять математику в бытовых ситуациях: измерять, сравнивать, считать время, деньги, длину, периметр, читать простые схемы и находить закономерности.

## Методический комментарий

Этот блок нужен, чтобы математика была живой. Ребенок должен не только решать примеры, но и понимать часы, деньги, линейку, длину, массу, объем, фигуры, маршрут по клеткам и простые закономерности.

Визуал здесь не украшение, а часть смысла. Без визуала ребенок может решать арифметику, но не научится уверенно читать реальные часы, линейку и схему.

## Состав блока

1. Величины.
2. Время.
3. Деньги.
4. Геометрия.
5. Закономерности.
6. Визуальные задания.

## Типы заданий

```text
text_task - текстовая задача
compare_units - сравнение величин
convert_units - перевод величин
clock_time - определить время по часам
time_interval - сколько времени прошло
time_order - раньше / позже
money_sum - сложить стоимость
money_change - найти сдачу
money_enough - хватит ли денег
money_compare - сравнить стоимость
geometry_shape - определить фигуру
geometry_segment - задача про отрезок
geometry_perimeter - найти периметр
geometry_broken_line - найти длину ломаной
grid_path - путь по клеткам
pattern_continue - продолжить ряд
pattern_rule - найти правило
odd_one_out - найти лишнее
ruler_measure - измерение по линейке
```

## Уровни сложности

```text
difficulty 1:
одно простое действие в пределах 100, без сложного рассуждения

difficulty 2:
2 шага, перевод единиц, сравнение после вычисления, время с минутами, периметр

difficulty 3:
2 действия, сдача, нехватка денег, сложное расписание, закономерность с правилом
```

## Визуальные типы

```text

```

## Общая логика выдачи

Одна тренировка: 8-10 заданий.

Рекомендуемая смесь:

- 2 задания на величины;
- 2 задания на время;
- 2 задания на деньги;
- 2 задания на геометрию;
- 1-2 задания на закономерности.

Если ребенок ошибся 2 раза в одной теме, эта тема чаще возвращается в следующих тренировках.

Если точность по теме выше 85%, можно подмешивать задания следующей сложности.

## ТЗ для программиста

### Задача

Реализовать интерактивный математический блок `Величины, время, деньги, геометрия и закономерности` для ребенка после 2 класса.

Блок должен поддерживать текстовые и визуальные задания.

### Основные сущности

Таблица `tasks`:


Статусы:

```text
mastered - ребенок решает уверенно
```

### Логика проверки ответа

Ответ считается правильным, если:

- для числовых ответов совпадает число;
- для ответов с единицей можно принимать число отдельно, если `unit` хранится в базе;
- для ответов `да / нет` принимается точное значение;
- для времени принимать нормализованный формат, например `10:45`.

Рекомендация: в базе `answer` хранить в нормализованном виде.

Примеры:

```text
answer: 20
unit: рубль
```

```text
answer: 10:45
unit: time
```

```text
answer: да
unit: boolean
```

### Отчет после тренировки

Показывать ребенку:

- сколько заданий выполнено;
- сколько правильных ответов;
- какая тема получилась лучше всего;
- какую тему нужно повторить.

Пример:

```text
Ты выполнила 10 заданий.
Правильно: 8 из 10.
Лучше всего получилось: деньги.
Нужно повторить: время и периметр.
```

### Отчет для родителя

- дата тренировки;
- количество заданий;
- точность в процентах;
- ошибки по темам;
- сложные задания;
- рекомендация, что повторить.

Пример:


## Главное правило

Блок должен учить ребенка применять математику в жизни.

Не просто:

```text
6 + 4 + 6 + 4
```

А:

```text
у прямоугольника стороны 6 см и 4 см. Найди периметр.
```

Не просто:

```text
50 - 32
```

А:

```text
у тебя 50 рублей, покупка стоит 32 рубля. Сколько сдачи?
```

Не просто:

```text
10:20 - 10:30
```

А:

```text
перемена началась в 10:20 и закончилась в 10:30. Сколько минут она длилась?
```
